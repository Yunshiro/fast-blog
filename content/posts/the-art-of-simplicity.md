Everyone talks about simplicity. Few practice it.

Adding features is easy. Removing them takes courage. Every line of code, every UI element, every dependency — each one carries a cost that compounds over time.

## The Hidden Costs

When you add something, you're not just adding it once. You're committing to:

- **Maintenance** — every feature must be updated when the platform changes
- **Cognitive load** — users (and future you) need to understand it
- **Interaction debt** — features interact in unexpected ways, creating bugs
- **Decision fatigue** — more options means more choices for every user

## What Simplicity Is Not

Simplicity isn't lack of features. A knife is simple — does exactly one thing, does it perfectly, lasts forever. Simplicity is focus.

It's also not laziness. Building something simple takes more effort than building something complex. You have to understand the problem deeply enough to know what to leave out.

## Start With "No"

The default answer to every feature request should be "no." Make the feature prove it deserves to exist. What problem does it solve? What happens if we don't build it? Can we solve the problem with what we already have?

The best features are the ones you never need to build.
